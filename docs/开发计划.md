## 附录 C：失败时的降级路径

### 如果 PoC-1 失败（GPU 缺页）

- **降级 1**：放弃 mlock 路径，改"权重预拷贝到 Metal buffer"（牺牲零拷贝优势）
- **降级 2**：缩小范围到 13B-30B 模型（24GB 够装权重）
- **降级 3**：放弃 FusionLLM，做 Ollama + MLX 配置优化工具

### 如果 PoC-4 失败（KV Cache SSD 一致性）

- **降级 1**：KV Cache 仍按 paged 内存管理，不落 SSD
- **降级 2**：只把"已完成会话"的 KV Cache 落 SSD，不做会话中换入换出
- **降级 3**：路径 c 失败 → 路径 b（70B + 4K 短上下文）

### 如果 Phase 3 不达标（70B 32K 跑不通）

- **降级 1**：定到 70B + 16K，作为"轻量路径 c"对外发布
- **降级 2**：把 KV Cache 分层作为可选 feature，关闭后回到 70B + 4K
- **降级 3**：路径 c 失败 → 路径 b（70B + 4K 短上下文）

### 如果工程师不能全职

- **降级 1**：延长时间到 6-7 个月（Phase 3）
- **降级 2**：砍掉 Phase 8-9 长上下文优化
- **降级 3**：找 part-time 工程师 + 老大补位
